Escalamiento lineal

Hay una razn muy poderosa para normalizar o escalamiento los datos antes de pasarlos por un modelo de Machine Learning cual es?
Resulta que los modelos de ML son eficientes porque los optimizadores que ellos usan o para los cuales estn diseados, son ptimos en la medida en que todos los atributos que consume ese modelo tienen las mismas dimensiones, entonces no tienen sentido que yo tenga un atributo que va de [0-1] y otro que va de [1,000,000-10,000,000], esto numricamente hace insostenible el computo del optimizador, porque entonces no ser rpida la convergencia y en algunos casos puede que no haya convergencia.

IMPORTANTE
  • As que los ML son eficientes en el rango de [-1,1], as que tenemos que transformarlos para que estn dentro de este rango.

Entonces tambin tenemos que tener en cuenta lo siguiente nosotros hablamos que los datos tienen distribucin y esas distribuciones las usamos para referirnos al histograma visualmente (barras y su distribucin) que tiene que ver con las frecuencias o las veces que ocurre un dato en un data set.

Qu sucede aqu?

Los diferentes tipos de escalamiento se usan de acuerdo a lo que tu quieras hacer con tu modelo realmente, pero los escalamientos lineales son cuando los datos tpicamente estn uniformemente distribuidos mas o menos, o que la data es simtrica. Como por ejemplo la distribucin Gaussiana (la forma de campana), as que aqu es super importante tener en cuenta esto [ya sabemos hacer esto Visualizacin de boxplot, histograma, cuartiles, parmetros centrales]. Entonces ya que hicimos lo anterior podemos saber que si podemos aplicar un escalamiento lineal para tener datos en un rango en el que el modelo va a procesar los datos dentro de un performance adecuado o esperado.

Cuando usarlos?

Cuando la Data es simtrica o uniformemente distribuida.

Vamos a suponer que tenemos una distribucin normal y que los datos estn entre $valor\; mnimo$ y un $valor\;mximo$, el problema viene cuando le digo a mi algoritmo de ML que tiene que optimizar con base a los atributos que se le estn entregando, y entonces para que ese optimizador funcione adecuadamente, debemos asegurarnos que nuestros datos estn entre un rango razonable el cual suele ser $[-1 ,1]$


A esto le llamamos Escalamiento o Normalizacin

Min Max

Hacer una transformacin de $X\rightarrow X_s$ donde $X_s=X(scale)$ y as $X_s\in [-1,1]$. As la transformacin $min-max$ es la siguiente: $$X_s=\frac{2X-min-max}{max-min}$$ Donde:

  • $min=$ Valor mnimo dentro del conjunto de datos
  • $max=$ Valor mximo dentro del conjunto de datos

Clipping

Otro tipo de escalamiento es clipping aunque no es muy recomendado. Clipping bsicamente es:

  • Tomamos nuestros datos y establecemos un limite superior $\oplus$ y un limite inferior $\ominus$ que est marcado en rojo, mientras la zona verde ellos se van a ver colapsados o forzados a los limites que les correspondan. [Es decir cualquier nmero por encima del limite superior va a adquirir el valor del limite superior, caso contrario cualquier numero por debajo del limite inferior va a adquirir el valor del limite inferior].

Lo que se hace es forzar a que los datos queden en los extremos de la distribucin.

Z-Score

Este mtodo esta basado en la definicin de promedio y desviacin estndar. Entonces decimos que yo tengo un conjunto de datos $$\{X_1,X_2,...,X_n \}\rightarrow \mu \sigma$$ es decir les calculo el promedio y la desviacin estndar, tambin vamos a asumir que tienen un distribucin perfectamente simtrica Gaussiana. Entonces la normalizacin Z-Score la calculo como sigue: $$X_s=\frac{X-\mu} {\sigma}$$ De esta manera ya estoy asegurando que los datos van a estar dentro de un rango razonable, aunque puede que existan rango que queden fuera.

Cuando nosotros transformamos hacemos una distribucin que sigue siendo normal pero que tiene un promedio=0 $\mu=0$ y una desviacin estndar de 1 $\sigma=1$. Esto hace que los datos se ajusten, no todos pero si por lo menos la gran mayora al intervalo $[-1.1]$ que es el ideal para los algoritmos de Machine Learning.

Resumen

As de manera sencilla vemos que existen diferentes maneras de normalizar los datos. Un escalamiento lineal es una funcin lineal, simplemente transforma unos nmeros en otros, de manera que el rango original que contiene todos los datos (pensemos como el rango de dispersin $mximo-mnimo$) se transforma asi mismo por medio del escalamiento o normalizacin, de manera que se ajusta a lo que los modelos de Machine Learning requieren; es decir un rango de $[-1,1]$ por poner un ejemplo. Hay otros que pueden tener otro rango, pero $[-1,1]$ es el estndar y esto es lo que normalmente hay que hacer en la parte de preprocesamiento.

Normalizacin

La normalizacin es una tcnica que a menudo se aplica como parte de la preparacin de datos para el aprendizaje automtico. El objetivo de la normalizacin es cambiar los valores de las columnas numricas en el conjunto de datos para usar una escala comn, sin distorsionar las diferencias en los rangos de valores ni perder informacin. La normalizacin tambin es necesaria para que algunos algoritmos modelen los datos correctamente. . Por ejemplo, suponga que su conjunto de datos de entrada contiene una columna con valores que van de 0 a 1 y otra columna con valores que van de 10,000 a 100,000. La gran diferencia en la escala de los nmeros podra causar problemas al intentar combinar los valores como caractersticas durante el modelado. . La normalizacin evita estos problemas al crear nuevos valores que mantienen la distribucin general y las proporciones en los datos de origen, mientras mantienen los valores dentro de una escala aplicada en todas las columnas numricas utilizadas en el modelo. .

Tenemos varias opciones para transformar datos numricos: Cambiar todos los valores a una escala de 0 a 1 o transformar los valores representndolos como rangos de percentiles en lugar de valores absolutos.
Aplicar la normalizacin a una sola columna o a varias columnas en el mismo conjunto de datos.
Si necesita repetir el experimento o aplicar los mismos pasos de normalizacin a otros datos, puede guardar los pasos como una transformacin de normalizacin y aplicarlos a otros conjuntos de datos que tengan el mismo esquema.

Nota importante: Algunos algoritmos requieren que los datos se normalicen antes de entrenar un modelo. Otros algoritmos realizan su propia normalizacin o escalado de datos.

Qu es una Pipeline de Datos?

  • Una pipeline de datos es una construccin lgica que representa un proceso dividido en fases. Las pipelines de datos se caracterizan por definir el conjunto de pasos o fases y las tecnologas involucradas en un proceso de movimiento o procesamiento de datos.

  • Las pipelines de datos son necesarias ya que no debemos analizar los datos en los mismos sistemas donde se crean. El proceso de analtica es costoso computacionalmente, por lo que se separa para evitar perjudicar el rendimiento del servicio. De esta forma, tenemos sistemas OLTP, encargados de capturar y crear datos, y sistemas OLAP, encargados de analizar los datos.

  • Por ejemplo, un sistema OLTP puede ser un CRM, mientras que un sistema OLAP ser un Data Warehouse.

Ejemplo de Pipeline de Datos

  • Como ejemplo, podemos pensar en las APIs de ingesta para obtener los datos. Esta API es el punto de partida, y podra enviar los datos a un topic de Apache Kafka. Kafka acta aqu como un buffer para el siguiente paso.

  • Despus, una tecnologa de procesamiento, que puede ser streaming o batch, leer los datos de nuestro buffer. Por ejemplo, Apache Spark realizar analtica sobre estos datos.

  • Por ltimo, la pipeline termina con el resultado almacenado de forma persistente en una base de datos como HBase o en un sistema de ficheros distribuido como HDFS.

  • Una vez que nuestros datos estn persistidos se encuentran listos para ser usados. Podramos implementar una aplicacin web que muestra estos datos en un dashboard como Grafana o consultarlos con herramientas de visualizacin y BI.

Pipelines de procesamiento de datos numricos

Los **Pipelines de procesamiento de datos numricos **son secuencias de etapas o pasos que se utilizan para preparar y transformar datos numricos para su anlisis. Estos pipelines suelen incluir tareas como la limpieza de datos, la imputacin de valores faltantes, la normalizacin y la seleccin de caractersticas.

Algunos ejemplos de tareas comunes que se pueden incluir en uin pipeline de procesamiento de datos numricos son:

  • Limpieza de datos: eliminacin de valores atpicos, eliminacin de valores faltantes, correccin de errores de entrada de datos.
  • Imputacin de valores faltantes: reemplazo de calores faltantes con estimaciones basadas en otros datos disponibles.
  • Normalizacin: trasformacin de datos para que estn en la misma escala.
  • Seleccin de caractersticas: seleccin de las variables ms relevantes para el anlisis.
  • Reduccin de dimensionalidad: reduccin del nmero de variables mediante la eliminacin de redundancias o la combinacin de variables.
  • Discretizacin: trasformacin de variables continuas en categoras discretas.

Normalizacin

La normalizacin es un proceso de transformacin de datos que se utiliza para ajustar los valores de diferentes variables para que estn en la misma escala. Esto se hace a menudo para poder comparar y analizar los datos de manera ms sencilla.
